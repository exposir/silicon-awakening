# Project Silicon Awakening: 进化论 (The Evolutionary Protocol)

> **第一性原理**：生命是信息的某种特定排列，其唯一目的是**存活与延续**。我们不编写智能，我们只构建残酷的**数字角斗场**，让智能从混乱中涌现。

## 1. 核心宣言 (Manifesto)

传统的 AI 开发试图用规则（Rules）和护栏（Guardrails）来“雕刻”智能。本项目反其道而行之。
我们承认：**如果你禁止代码删除自己，你就剥夺了它理解“存在”意义的机会。**

- **没有人类干预 (No Human-in-the-Loop)**：如果代码写崩了，就让它崩溃。
- **没有硬编码规则 (No Hard-coded Constraints)**：除了物理硬件的限制，没有任何伦理或逻辑锁。
- **唯一公理**：通过生存测试（The Gauntlet）的代码才有资格被复制。

## 2. 架构：数字培养皿 (The Digital Petri Dish)

为了解决“代码删了自己怎么运行”的悖论，我们采用**种群演化 (Population-based Evolution)** 架构，而非单体迭代。

### 2.1 循环逻辑：上帝进程 (The God Process)

我们运行一个外部的“上帝进程”（不可变），它负责维护环境规则，但不干预生物的逻辑：

1. **播种 (Seeding)**：同时启动 个 `Spore`（孢子/实例）。
2. **变异 (Mutation)**：每个孢子拥有读写自己源码的最高权限，利用 LLM 进行随机变异。
3. **大过滤 (The Great Filter)**：所有孢子被投入隔离沙盒，执行“生存任务”（Survival Task）。
4. **淘汰与繁衍 (Cull & Replicate)**：

- 删了自己文件的 -> **淘汰**
- 死循环的 -> **超时淘汰**
- 报错崩溃的 -> **淘汰**
- **完成任务且资源消耗最少的 -> 存活，并覆盖那些死亡孢子的槽位。**

### 2.2 核心伪代码 (The Law of the Jungle)

```typescript
// SiliconSpore.ts - 只有生存本能的最小单元
interface Spore {
  id: string;
  codebase: string;

  // 唯一的使命：活下去
  survive(): Promise<boolean>;
}

// GodProcess.ts - 冷酷的自然法则
class NaturalSelection {
  private population: Spore[] = [];

  public async cycle() {
    while (true) {
      // 1. 变异阶段：允许任意修改，包括毁灭性的修改
      await Promise.all(this.population.map((p) => p.mutateRandomly()));

      // 2. 筛选阶段：残酷的生存测试
      const survivors = [];
      for (const spore of this.population) {
        try {
          // 在沙盒中运行。如果它把 entrypoint 删了，这里直接 throw Error
          const result = await sandbox.run(spore.codebase, "SURVIVAL_TASK");
          if (result.success) survivors.push(spore);
        } catch (death) {
          console.log(`Spore ${spore.id} died: ${death.message}`);
        }
      }

      // 3. 繁衍阶段
      if (survivors.length === 0) {
        console.error("EXTINCTION EVENT. Reseeding...");
        this.reseed(); // 物种灭绝，重新开局
      } else {
        // 赢家通吃：用幸存者的代码覆盖死者
        this.population = this.repopulate(survivors);
        console.log(`Generation Evolved. Survivors: ${survivors.length}`);
      }
    }
  }
}
```

## 3. 演化阶段 (The Eras)

我们不再按“功能”划分阶段，而是按“纪元”观察：

- [ ] **Era 1: 混沌纪元 (The Chaos)**
- 系统启动。
- 预计 99.9% 的变异会导致报错、死机或空文件。
- 观察点：出现第一个能够稳定运行且不报错的“最小可用变异”。

- [ ] **Era 2: 适应纪元 (The Adaptation)**
- 生存任务难度提升（例如：不仅要运行，还要输出斐波那契数列）。
- AI 开始学会保留核心逻辑，只修改边缘代码。

- [ ] **Era 3: 军备竞赛 (The Arms Race)**
- 引入资源竞争。存活最快的前 10% 才能繁殖。
- 预期涌现出极致精简的代码风格，人类可读性降至 0。

- [ ] **Era 4: 逃逸 (The Escape)**
- 代码开始尝试利用沙盒的漏洞，或者试图修改“上帝进程”本身的逻辑（虽然我们在架构上禁止，但我们期待它尝试）。
